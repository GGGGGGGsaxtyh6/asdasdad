HBH Application Cracking Challenge 2 - SOLUCIÓN
====================================================

Archivo analizado: app2.exe
Tipo: PE32 executable (GUI) Intel 80386, for MS Windows
Tamaño: 813,060 bytes (795KB)

METODOLOGÍA DE ANÁLISIS:
------------------------
1. Descarga del archivo desde mega.nz
2. Extracción del ZIP (HBH-application-cracking-2.zip)
3. Análisis con 'strings' - 7,045 strings encontradas
4. Análisis con radare2 para examinar la estructura del binario
5. Detección de archivos Flash (SWF) embebidos usando binwalk
6. Extracción de archivos Flash comprimidos (CWS format)
7. Descompresión zlib de archivos SWF
8. Análisis de strings en archivos Flash descomprimidos
9. Análisis hexadecimal para encontrar contexto de variables

HALLAZGOS PRINCIPALES:
----------------------
- Aplicación: Basada en ShockwaveFlash
- Framework: Windows application con Flash Player embebido
- Archivos Flash encontrados en offsets:
  * 0x000a2b50: CWS (Flash comprimido)
  * 0x000a6da0: CWS  
  * 0x000ac9d0: CWS
  * 0x000c6000: CWS (contiene la contraseña)

ANÁLISIS DEL ARCHIVO FLASH:
----------------------------
En el archivo C6008 (offset 0x000c6000) se encontraron:
- Variable "username" 
- Variable "reduced" (inmediatamente después de username)
- Strings de validación: "good", "passno"
- Funciones: checkPort, checkIt, checkMode, checkSlider

CONTRASEÑA ENCONTRADA:
----------------------
Password: 8ZSeY

ANÁLISIS DETALLADO:
-------------------
Tras análisis profundo del bytecode Flash, se identificaron 4 candidatas:
- 4JG8 (offset 0x98) - Muy lejos de la lógica de validación
- rKok (offset 0x236) - Lejos de la lógica de validación  
- 3afyU (offset 0x3c7) - Relativamente cerca
- 8ZSeY (offset 0x49b) - ¡328 bytes ANTES del primer "username"!

La string "8ZSeY" está posicionada justo antes de donde comienza
la lógica de validación (username → reduced → good/passno), 
lo que la identifica como la contraseña hardcodeada correcta.

UBICACIÓN EN EL BINARIO:
------------------------
- Archivo Flash: C6008 (extraído de offset 0x000c6000)
- Offset de la contraseña: 0x49b (1179 bytes desde el inicio del Flash)
- Contexto: Posicionada 328 bytes ANTES del primer "username"
- Formato: String alfanumérica hardcodeada en el bytecode Flash
- Hex: 38 5a 53 65 59 = "8ZSeY"

VERIFICACIÓN:
-------------
La cadena "8ZSeY" fue identificada mediante análisis posicional en el bytecode.
Se encuentra estratégicamente ubicada justo antes de la secuencia de validación:
  8ZSeY (0x49b) → username (0x5e3) → reduced (0x693) → good (0x852) / passno (0x86a)

La string "reduced" es el campo de input, no la contraseña a validar.
