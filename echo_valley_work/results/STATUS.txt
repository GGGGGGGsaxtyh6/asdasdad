ANÁLISIS COMPLETO:
------------------

1. VULNERABILIDAD IDENTIFICADA:
   - Format String Attack en línea 39: printf(buf);
   - Buffer de 100 bytes
   - Función print_flag() disponible en offset 0x1269 desde base

2. PROTECCIONES DEL BINARIO:
   - PIE: SÍ (direcciones aleatorias)
   - FULL RELRO: SÍ (GOT es read-only)
   - Stack Canary: SÍ  
   - NX: SÍ

3. ESTRATEGIAS INTENTADAS (SIN ÉXITO):

   a) Sobrescribir GOT de printf/fflush/puts:
      - Intentado con múltiples funciones
      - NO funciona debido a FULL RELRO
      - Confirmado tanto local como remotamente
      
   b) Sobrescribir Return Address en el stack:
      - Calculé múltiples offsets basados en stack leaks
      - Probé offsets desde -0x18 hasta +0xc8 desde stack20/stack22/stack24
      - Intenté con escrituras de 2 bytes (hn) y byte-por-byte (hhn)
      - Ningún offset causó ejecución de print_flag()
      - Siempre retorna normalmente con "The Valley Disappears"
      
   c) Escritura directa en posición 21:
      - Posición 21 muestra el VALOR del return address
      - Intenté sobrescribir usando %21$hn
      - Solo escribe EN la posición del stack, no en la dirección apuntada

4. INFORMACIÓN DEL STACK LEAKEADA:
   - Posición 1: libc address
   - Posición 20: stack pointer (~0x7fff...de0)
   - Posición 21: return address VALUE
   - Posición 22: stack pointer (~0x7fff...e80)
   - Posición 24: stack pointer
   - Diferencia pos22-pos20: ~0xa0 bytes

5. DIRECCIONES CALCULADAS:
   - Base del binario: leak_pos21 - 0x1413
   - print_flag(): base + 0x1269
   - GOT printf: base + 0x3fa0
   - GOT fflush: base + 0x3fb8
   - GOT puts: base + 0x3f88

6. PAYLOADS PROBADOS:
   - Escrituras de 3 shorts (6 bytes totales)
   - Escrituras de 6 bytes individuales
   - Escrituras de 2 bytes bajos solamente
   - Payloads de 58-114 bytes (todos < 100 para evitar overflow)
