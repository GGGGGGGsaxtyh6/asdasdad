========================================

1. ESTRUCTURA DE LA BASE DE DATOS:
- Tabla: users
- Columnas: user_id, user_name, user_privileges, user_password
- user_id=0 => 'admin', privileges=0 (all), password=$flag
- user_id 1-24 => usuarios normales, privileges=1 (no)

2. FLUJO DEL CÓDIGO:
   a) $_GET['ids'] debe ser un string
   b) Máximo 70 caracteres de longitud
   c) Se hace explode() por comas
   d) Cada elemento se convierte a (int)
   e) Si el valor es < 1, se elimina con unset()
   f) Se hace implode() de nuevo
   g) Se construye query: WHERE (user_id in ($selector))

3. VULNERABILIDAD:
   El problema está en el uso de unset() en un array. Cuando usas unset() en PHP,
   NO se reindexan los elementos del array. Esto significa que si tenemos:
   
   Array: [0 => '1', 1 => '0', 2 => '2']
   Después de convertir a int: [0 => 1, 1 => 0, 2 => 2]
   Después de unset($tmp[1]) porque 0 < 1: [0 => 1, 2 => 2]
   
   Ahora, cuando hacemos implode(',', $array) con un array que tiene índices no continuos,
   implode() SOLO usa los valores, pero el array_unique() mantiene las claves originales.
   
   Si podemos inyectar algo que no sea filtrado correctamente, podemos inyectar SQL.

4. ESTRATEGIA DE EXPLOTACIÓN:
   El punto clave es que después del filtrado, el $selector se inserta directamente
   en la query SIN escapar. Si logramos que después del procesamiento quede algo
   que no sea solo números y comas, podemos inyectar SQL.
   
   El truco está en que implode() y array_unique() no limpian los valores
   completamente. Si enviamos valores que al convertirse a (int) sean >= 1,
   pasarán el filtro.
   
   PERO ESPERA: El casting a (int) convierte cualquier string a número.
   "1) OR user_id=0--" => (int) = 1
   
   Así que después del (int), todos los valores maliciosos se limpian...
   
   NUEVO ENFOQUE: Analizar el comportamiento de explode/implode/array_unique
   con valores especiales.

   Mirando más de cerca: después de unset(), el array puede tener keys no secuenciales.

   Pero wait... todos los valores son (int), así que siempre serán números.
   
   ¡AJÁ! El truco está en el hecho de que podemos usar NOTACIÓN CIENTÍFICA o HEXADECIMAL.
   En PHP, (int)"1e1" = 10, pero si de alguna forma el string original sobrevive...
   
   NO, eso no funciona porque el casting a int destruye todo.

   En PHP, cuando haces unset() en elementos de un array y luego implode(),
