# Reporte de Progreso - Reto SUS

## Información del Reto
- Nombre: SUS - Single User Storage
- Servidor: svc.pwnable.xyz:30011
- Archivo: challenge_32.gz

## Análisis Completado

### 1. Reconocimiento Inicial
- Binary: ELF 64-bit x86-64
- Sin PIE (direcciones fijas)
- Stack canary presente
- NX habilitado (stack no ejecutable)

### 2. Funcionalidades del Programa
- Create user: aloca 32 bytes en heap, lee nombre y edad
- Print user: imprime datos del usuario
- Edit user: modifica nombre y edad
- Exit: sale del programa

### 3. Función Objetivo
- win() en 0x400b71: ejecuta "cat flag"
- Nunca llamada en flujo normal

### 4. Vulnerabilidad Identificada
- Use-After-Return en variable global `cur` (0x602268)
- cur apunta a [rbp-0x1060] de create_user
- Después del retorno, esta memoria es inválida
- edit_usr escribe en [cur+0x48] = [rbp_create - 0x1018]

### 5. Capacidades de Escritura
- Nombre: 32 bytes en heap (via create/edit)
- Edad: 4 bytes int32 en [cur+0x48] (via create/edit)
