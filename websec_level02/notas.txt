ANÁLISIS DEL RETO WEBSEC LEVEL02
==================================

VULNERABILIDAD:
- SQL Injection en el parámetro user_id
- Query: SELECT id,username FROM users WHERE id=' + $injection + ' LIMIT 1

FILTROS APLICADOS:
- preg_replace con flag 'i' (case insensitive)
- Palabras bloqueadas: union, order, select, from, group, by

BYPASS STRATEGY:
El filtro usa preg_replace solo UNA VEZ. Si duplicamos las keywords, se eliminará solo una parte:
- "selselectect" -> "select" (después del filtro)
- "uniunionon" -> "union" (después del filtro)
- "frfromom" -> "from" (después del filtro)

OBJETIVO:
Extraer la flag usando UNION SELECT bypass
